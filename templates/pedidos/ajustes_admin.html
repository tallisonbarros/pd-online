Checklist

O que o admin precisa fazer

  1. Criar ou selecionar o projeto no Google Cloud usado pela operação do app.
  2. Habilitar Maps JavaScript API, Geocoding API e Places API / Places API (New) no mesmo projeto.
  3. Gerar uma browser key com restrição por domínio para localhost, Render e domínio final da operação.
  4. Preencher GOOGLE_MAPS_API_KEY no ambiente local e na hospedagem.
  5. Opcionalmente ajustar GOOGLE_MAPS_LANGUAGE e GOOGLE_MAPS_REGION se o projeto operar fora do padrao pt-BR / BR.
  6. Reiniciar a aplicação depois de salvar as variáveis, para o checkout carregar o novo provedor.

Depois da chave ativa, o checkout passa a usar Google Maps como fonte principal da rua, bairro e reverse geocode. O frete continua sendo calculado no backend pelas coordenadas confirmadas no mapa.
